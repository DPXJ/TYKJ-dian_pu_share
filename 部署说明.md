# 部署到 GitHub Pages 说明

## 方式一：使用批处理脚本（推荐）

直接双击运行 `deploy.bat` 文件，脚本会自动执行以下操作：
1. 检查 Git 状态
2. 添加所有更改的文件
3. 提交更改
4. 推送到 GitHub

## 方式二：手动执行命令

在项目根目录打开命令行，依次执行以下命令：

```bash
# 1. 检查当前状态
git status

# 2. 添加所有更改的文件
git add .

# 3. 提交更改
git commit -m "feat: 添加除草成效分析报告页面及分享功能

- 新增除草成效分析报告页面 (weedControlAnalysisReport)
- 实现完整的分享功能（微信、朋友圈、抖音、生成图片、复制链接）
- 优化按钮点击事件，点击除草成效分析按钮直接跳转到报告页面
- 添加报告页面样式和分享海报样式"

# 4. 推送到 GitHub
git push origin main
```

## 配置 GitHub Pages

推送代码后，需要在 GitHub 上配置 Pages：

1. **打开 GitHub 仓库页面**
2. **点击 Settings（设置）标签**
3. **在左侧菜单中找到 Pages（页面）**
4. **在 Source（源）部分，选择 "GitHub Actions"**
5. **保存设置**

## 自动部署

配置完成后，GitHub Actions 会自动运行部署流程：
- 可以在仓库的 **Actions** 标签中查看部署进度
- 部署通常需要 1-2 分钟
- 部署完成后，网站将在以下地址可用：
  ```
  https://[您的GitHub用户名].github.io/[仓库名]/
  ```

## 注意事项

1. **仓库可见性**：免费账户需要将仓库设置为公开（Public）才能使用 GitHub Pages
2. **分支名称**：确保主分支是 `main`（不是 `master`）
3. **首次部署**：第一次部署可能需要几分钟时间
4. **自动更新**：之后每次推送到 `main` 分支都会自动重新部署

## 检查部署状态

1. 在 GitHub 仓库页面，点击 **Actions** 标签
2. 查看最新的工作流运行状态
3. 如果显示绿色 ✓，说明部署成功
4. 如果显示红色 ✗，点击查看错误详情
